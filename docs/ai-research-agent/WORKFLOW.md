## 典型用户场景

### 场景 1：从头开始研究

```
1. 启动 OpenCode
2. /research init <课题名称>
3. 输入研究问题: "如何提升Transformer的效率？"
4. Agent 分析并创建初始原子
5. 用户编辑/补充原子内容
6. 定义验证协议
7. 建立原子依赖关系
8. 查看知识图谱
9. 触发验证
10. 分析结果
11. 迭代优化
```

### 场景 2：已有代码基础

```
1. 进入现有代码目录
2. /research init <分析任务>
3. 输入: "分析这段代码的复杂度"
4. Agent 自动扫描代码创建观察原子
5. 用户补充假设
6. 定义验证实验
7. 执行验证
8. 归档结果
```

### 场景 3：验证他人假设

```
1. /research import <他人原子JSON>
2. 查看导入的原子和关系
3. 补充自己的验证协议
4. 执行实验
5. 给出验证结论
```

### 场景 4：从文献构建研究网络

```
1. 进入课题目录
2. /research paper import arXiv:2007.04758
3. 系统自动抓取文献元数据和摘要
4. /research paper analyze <paper-id>
5. Agent 分析文献内容，提取关键发现
6. 自动生成原子并建立引用关系
7. 导入更多相关文献
8. 系统自动建立文献间的引用关系
9. 查看完整的研究网络图谱
10. 基于文献发现提出自己的假设
```

## 核心命令流

```bash
# ===== 课题管理 =====
research init <name>              # 初始化新课题
research switch <id>              # 切换课题
research list                     # 查看所有课题
research delete <id>              # 删除课题

# ===== 原子操作 =====
research atom create             # 创建原子（交互式）
research atom edit <id>           # 编辑原子
research atom show <id>           # 查看详情
research atom list                # 列出原子
research atom delete <id>         # 删除原子

# ===== 代码项目管理 =====
research code add <name> <path>  # 添加代码项目 (本地路径或git)
research code list                # 列出代码项目
research code show <id>          # 查看代码项目详情
research code version <id> <ver> # 创建版本快照
research code delete <id>         # 删除代码项目

# ===== 文献操作 =====
research paper import <source>   # 导入文献 (arXiv/DOI/URL/PDF)
research paper list               # 列出文献
research paper analyze <id>       # 分析文献生成原子
research paper show <id>          # 查看详情
research paper delete <id>        # 删除文献

# ===== 关系操作 =====
research rel add <src> <tgt> <type>  # 添加关系
research rel remove <id>             # 删除关系
research rel graph                   # 查看依赖图

# ===== 验证执行 =====
research validate <atom-id>      # 验证原子
research validate --all            # 验证所有就绪原子
research status                   # 查看验证状态

# ===== 可视化 =====
research graph                    # 打开图谱 UI
research graph --focus <id>       # 聚焦某个原子
```

    │         │
    │         ▼

## 验证工作流（详细）

```
原子 <status=pending, dependencies 已满足>
    │
    ▼
/research validate <atom-id>
    │
    ▼
┌─────────────────────────────────────┐
│  Agent 读取原子内容                  │
│  • validation.protocol              │
│  • validation.metrics               │
│  • validation.experimentConfig      │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  生成实验代码                        │
│  • 解析验证协议                      │
│  • 选择合适的代码模板                 │
│  • 填充实验参数                      │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  执行实验                            │
│  • 连接远程服务器                    │
│  • 创建实验环境                      │
│  • 运行代码                          │
│  • 实时回传日志                      │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  收集结果                            │
│  • stdout/stderr                    │
│  • 性能指标                          │
│  • 生成的文件                        │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  分析结果                            │
│  • 对比预期指标                      │
│  • 判定验证结果 (Pass/Fail)          │
│  • 生成 evidence                     │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  归档                                │
│  • 更新 atom.status                 │
│  • 添加 evidence                    │
│  • 记录实验日志                      │
│  • 清理实验环境                      │
└─────────────────────────────────────┘
    │
    ▼
   Done
```

## 文献分析工作流（详细）

```
文献导入
    │
    ▼
┌─────────────────────────────────────┐
│  1. 获取文献内容                     │
│     • arXiv API 抓取                │
│     • DOI 解析 + 出版商API          │
│     • URL 网页抓取                  │
│     • PDF 解析 (本地文件)           │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  2. AI 分析文献                      │
│     • 提取关键发现 (Findings)        │
│     • 识别研究方法 (Methods)         │
│     • 梳理假设前提 (Hypotheses)      │
│     • 找出观察结果 (Observations)    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  3. 生成原子                         │
│     • 观察原子 (从文献发现提取)      │
│     • 方法原子 (从技术方法提取)      │
│     • 假设原子 (从研究假设提取)       │
│     • 标注文献来源                   │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  4. 建立关系                         │
│     • 原子引用文献来源               │
│     • 文献间的引用关系               │
│     • 与现有原子建立关联              │
└─────────────────────────────────────┘
    │
    ▼
   Done (文献状态: analyzed)
```

## 实验与代码链接工作流

```
创建实验
    │
    ▼
┌─────────────────────────────────────┐
│  1. 创建实验目录                     │
│     experiments/exp_xxx/           │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  2. 关联代码项目                     │
│     config.json 写入:               │
│     {                               │
│       "code": {                     │
│         "baseline": "code/baseline@v1.0.0", │
│         "proposed": "code/proposed@v1.0.0"  │
│       }                             │
│     }                               │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  3. Agent 执行实验                   │
│     • 读取 config.json 获取 code 索引│
│     • 从 code/ 目录读取对应代码     │
│     • 在代码目录执行实验            │
│     • 记录输出和指标                │
└─────────────────────────────────────┘
    │
    ▼
   Done (实验完整归档，代码统一管理)
```
